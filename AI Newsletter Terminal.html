04Claude Code 技巧
$频繁用 /clear 保持上下文清洁
Claude 上下文达 90% 时输出质量显著下降，关键指令会被淹没。每次开始新任务前用 /clear 清空对话，是提升输出质量最有效的习惯。
$用 Skills 处理重复性任务
把每周重复的任务（代码审查模板、报告格式）写成 Skill 文件，按需加载而非写入 CLAUDE.md，有效减少每次会话的上下文占用。
$Esc 停止不丢上下文，双击可回溯检查点
按 Esc 立即停止 Claude 正在执行的动作而不丢失上下文，可立即重新引导。双击 Esc 或输入 /rewind 进入历史检查点菜单，支持回滚代码和对话。
$给 Claude 验证方法让它自迭代
最重要技巧：给 Claude 一个可验证输出的方法（测试套件、linter、构建命令），它会持续迭代直到通过，成功率从 33% 提升至 80%+。
$/install-github-app 自动审查 PR
安装后 Claude Code 会自动审查所有新 PR，无需手动触发。高产工程师与普通用户的差距，往往在于能否让 AI 自动化处理重复审查工作。
