## 四、💡 Claude Code 使用技巧

### 技巧 1：用 Hooks 替代 CLAUDE.md 做强制性规范
CLAUDE.md 中的指令是"建议性"的（遵守率约 70%）。对于必须 100% 执行的操作（代码规范、安全扫描），改用 `.claude/settings.json` 中配置 **Hooks**，在文件写入前后自动触发脚本。

### 技巧 2：`/ultraplan` — 释放本地上下文做深度架构规划
2026 年 4 月新上线命令。将复杂规划任务转交给基于 Opus 4.6 的云端 session（最长 30 分钟），本地终端保持空闲可继续工作。适用于大型重构、系统架构设计。

### 技巧 3：Worktrees + 并行 Session 加速开发
```bash
claude --worktree feature-auth
claude --worktree feature-payments
```
每个 worktree 拥有独立上下文，互不干扰，可同时推进多个功能。另起一个全新 session 做 Code Review，避免 Claude 偏向自己刚写的代码。

### 技巧 4：用 Skills 模块化团队知识库
在 `.claude/skills/` 目录下建立技能文件夹，包含规范文档、校验脚本和 **Gotchas 章节**（记录实际踩过的坑）。结合 `@imports` 语法按需加载，保持主上下文精简。

### 技巧 5：Prompt Caching 重要变化（2026 必读）
⚠️ Anthropic 已将 Prompt Cache TTL 从 **60 分钟缩短至 5 分钟**，导致许多生产环境 API 成本上涨 30–60%。应对：将静态内容（system prompt、工具定义）放最前并打 `cache_control` 标记；多步骤 tool use 工作流使用 1 小时缓存时长维持跨步骤 cache hit。
